# 예약 플로우

## 문의 → 예약 → 송객 단계
```
[1] 문의 발생
   ├─ OTA 예약 (자동)
   ├─ DM/이메일 문의
   └─ 현장 워크인

[2] 응대 / 견적
   ├─ 영문 자동 응답 + 사람 응대
   ├─ 일정·인원·옵션 확정
   └─ 결제 링크 발송 (또는 OTA 자체 결제)

[3] 예약 확정
   ├─ 확정 이메일 (영문, 위치/픽업/준비물 안내)
   ├─ 캘린더에 일정 등록
   └─ 파트너에게 인원·시간 통보

[4] 사전 리마인드 (D-1)
   ├─ 날씨/운항 확인
   ├─ 픽업 시간 재확인
   └─ 준비물 안내

[5] 당일 송객 / 가이드
   ├─ 픽업 (해당 시)
   ├─ 안전 브리핑 (영문)
   ├─ 체험 진행
   └─ 사진/영상 공유

[6] 사후
   ├─ 리뷰 요청 (Google, OTA)
   ├─ UGC 사용 동의 요청
   └─ 정산 집계 (파트너)
```

## 응답 SLA
- DM/이메일: 영업시간 1시간 내, 영업시간 외 24시간 내
- OTA 자동 확정 / 수동 확정 구분 정책 결정 필요

## 도구 (초기)
- 캘린더: Google Calendar (전용 캘린더)
- 메시지: 인스타 DM / 이메일 / OTA 인박스
- 파이프라인 트래킹: `pipeline.md` (또는 노션·스프레드시트)
- 결제: OTA 자체 결제 / Stripe / 토스페이먼츠 (TBD)
